## Key concepts

| Concept               | What it means                                                                                                                      |
| --------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| **Context as Code**   | `terse generate` compiles your connected integrations into TypeScript — no runtime discovery of IDs, channels, or lists            |
| **Workflow**          | A TypeScript automation with triggers, skills, and an `onTrigger` handler, deployed via `terse deploy`                             |
| **Trigger**           | The event (e.g. `Attio.onRecordCreated()`) or schedule (e.g. `Schedule.cron()`) that starts a workflow                             |
| **Skill**             | An integration capability available to the workflow (e.g. `Apollo.skill()`, `Slack.skill()`)                                       |
| **Generated helpers** | Typed exports in `src/terse.generated.ts` — resource constants, trigger builders, skill constructors, and `agent.tools.*` wrappers |
